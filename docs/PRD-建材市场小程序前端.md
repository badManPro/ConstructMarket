## 2. 产品概述

### 2.1 项目背景
面向建材市场交易场景，建设一款以商品浏览、选型、加购、下单、支付、订单查看和售后服务为核心的小程序。首版以微信原生小程序落地，只考虑前端用户体验与页面流程，不包含后端系统、运营后台、接口设计和真实支付接入。

### 2.2 产品定位
- 产品定位：工程/企业采购型建材商城小程序
- 交易模式：以直接下单支付为主，兼顾建材行业对规格、起订量、开票、地址管理和售后服务的诉求
- 设计原则：
  - 首页要快速导流到搜索、分类和活动商品
  - 商品详情要突出规格、价格、起订量和购买动作
  - 个人中心要覆盖采购后服务能力
  - 所有数据先使用 Mock，本地即可演示完整链路

### 2.3 目标用户
- 工程采购人员
- 小型施工团队负责人
- 建材市场门店补货人员
- 有明确型号、规格、品牌诉求的 B 端采购用户

### 2.4 核心使用场景
1. 用户在首页通过搜索、分类导航或活动推荐进入商品详情并购买
2. 用户按建材品类进行选型，在搜索结果页通过排序和筛选快速定位商品
3. 用户在购物车统一修改数量、确认商品后进入结算和支付
4. 用户在个人中心查看订单、发票、地址、优惠券和个人资料
5. 用户在资讯模块阅读行业新闻、产品知识和装修指南
6. 用户在客服系统中进行在线咨询、查看常见问题或提交投诉建议

### 2.5 产品目标
- 建立完整的小程序前端交易闭环：浏览 -> 选型 -> 加购 -> 结算 -> 支付 -> 订单查看
- 将脑图中的核心模块全部沉淀为可开发页面和交互
- 输出能直接指导页面拆分、状态设计、Mock 数据准备的 PRD

### 2.6 V1 范围
- 首页设计
- 建材资讯
- 购物车
- 订单与支付结果
- 我的（个人中心）
- 发票开具相关页面
- 收货地址、收藏夹、优惠券、个人信息
- 客服系统
- 支付方式选择与支付结果模拟

### 2.7 非范围项
- 后端接口定义与数据库设计
- 管理后台、商家后台、CMS
- 实时聊天服务端能力
- 真实支付 SDK 接入
- 库存实时校验、ERP 对接、审批流、消息推送
- 竞品参考稿中出现但脑图未明确要求的“消息中心、订单审批、我的清单、导出报价单”等能力

- `筛选侧栏抽屉`
  - 出现场景：搜索结果页
  - 作用：价格区间、起订量区间、排序偏好、属性筛选
- `规格选择弹层`
  - 出现场景：商品详情页
  - 作用：选择规格、颜色、型号、数量
- `发票信息弹层`
  - 出现场景：结算页、发票中心页
  - 作用：选择电子/纸质发票，填写开票信息
- `支付方式选择区`
  - 出现场景：结算页
  - 作用：切换微信支付、支付宝支付、银联支付
- `删除确认弹窗`
  - 出现场景：购物车页、地址页、收藏夹页
  - 作用：二次确认删除操作

## 4. 前端范围说明

### 4.1 仅覆盖前端
- 只定义页面、状态、导航、交互、异常态和 Mock 数据结构
- 不定义真实接口路径、请求参数、数据库字段
- 所有数据本地模拟，可通过本地 JSON、TS 常量或页面内 Mock 对象提供

### 4.2 页面状态规范
每个页面至少支持以下状态：
- `loading`：骨架屏或局部加载中
- `normal`：正常渲染
- `empty`：无数据
- `error`：请求失败或 Mock 异常
- `offline`：网络异常占位

### 4.3 本地存储建议
- 购物车商品、收藏状态、默认地址、最近搜索词、已选优惠券、草稿发票信息、客服会话消息可使用本地缓存
- 支付结果和订单状态流转可通过 Mock Store 统一维护

## 6. 页面详细需求

### 6.1 首页
- 页面目标：承接首屏流量，快速分发到搜索、分类、活动商品、热门商品和资讯内容
- 页面入口：小程序默认首页；支付结果返回首页； TabBar 首页
- 核心模块：
  - 搜索入口
  - 轮播图/装修图
  - 商品分类导航
  - 活动商品推荐
  - 热门商品推荐
  - 建材资讯入口
- 主要交互：
  - 点击搜索框进入搜索结果页并回填关键词
  - 点击轮播图进入活动专题或商品详情
  - 点击分类图标跳转选型/分类页并定位到对应类目
  - 点击商品卡片进入商品详情
  - 点击资讯入口进入建材资讯页
- 空态/异常态：
  - 活动商品为空时展示默认占位卡片
  - 首页接口异常时展示重试按钮和骨架屏降级
- 页面跳转：
  - -> 搜索结果页
  - -> 选型/分类页
  - -> 商品详情页
  - -> 建材资讯页
- Mock 字段：
  - `banners`
  - `categoryNav`
  - `campaignProducts`
  - `hotProducts`
  - `articleEntrances`

### 6.2 选型/分类页
- 页面目标：帮助用户基于类目进行建材选型和浏览
- 页面入口：TabBar 选型；首页分类导航
- 核心模块：
  - 左侧一级类目列表
  - 右侧二级类目/商品宫格
  - 当前类目标题
  - 返回搜索结果的链路入口
- 主要交互：
  - 点击左侧一级类目刷新右侧内容
  - 点击二级类目进入搜索结果页并携带类目参数
  - 点击商品卡片进入商品详情页
- 空态/异常态：
  - 某一级类目下无内容时显示“该分类暂无商品”
  - 类目加载失败时支持重试
- 页面跳转：
  - -> 搜索结果页
  - -> 商品详情页
- Mock 字段：
  - `rootCategories`
  - `subCategories`
  - `categoryProducts`
  - `selectedCategoryId`

### 6.3 搜索结果页

## 7. 关键业务链路

### 7.1 浏览与搜索链路
1. 首页输入关键词
2. 进入搜索结果页
3. 调整排序与筛选
4. 进入商品详情页

### 7.2 分类选型链路
1. 首页点击分类导航或 TabBar 进入选型页
2. 选择一级类目和二级类目
3. 进入搜索结果页或直接打开商品详情

### 7.3 交易链路
1. 商品详情点击加入购物车或立即购买
2. 购物车修改数量并结算
3. 结算页选择地址、发票、优惠券和支付方式
4. 提交订单
5. 模拟支付成功或失败
6. 查看订单详情

### 7.4 内容链路
1. 首页进入建材资讯页
2. 切换资讯分类
3. 查看资讯详情
4. 通过相关推荐继续阅读

### 7.5 服务链路
1. 我的页进入客服系统首页
2. 进入在线咨询、常见问题或投诉建议
3. 在订单详情页或商品详情页也可快捷联系客服

## 8. 页面状态与异常态规范

### 8.1 通用异常态
- 列表为空：展示空态插画、说明文案和推荐去向
- 加载失败：展示错误提示和重试按钮
- 网络异常：展示离线提示与刷新动作

### 8.2 关键异常场景
- 搜索无结果：展示“换个关键词试试”与热门分类
- 购物车为空：展示“去首页逛逛”
- 无收货地址：强引导新增地址
- 无优惠券：展示说明文案，不阻断交易
- 支付失败：停留结果页，展示失败原因与重新支付
- 发票记录为空：展示“暂无开票记录”
- 在线咨询无历史消息：展示欢迎语和快捷问题
