# ConstructMarket UI Reference Guideline

这份文档用于固定当前项目的 UI 设计约束，方便后续任务自动延续同一套视觉方向。

## 双来源规则

- 功能、页面边界、路由和状态流转：以 `docs/PRD-建材市场小程序前端.md` 为准
- 主题、布局、信息密度、卡片形态、按钮样式、底部导航、弹层视觉：以 `docs/pencil/miniapp.pen` 和 `docs/pencil/*.jpg` 为主

结论：
- 不再把 `docs/pencil` 仅视为“补充参考”
- 从当前阶段开始，`docs/pencil` 是 UI 设计主来源
- PRD 仍然是功能主来源，若两者冲突，按“功能遵循 PRD、视觉遵循 pencil”处理

## 当前视觉基调

- 平台感：接近成熟建材电商微信小程序，而不是展示型原型页面
- 背景：浅灰底 `#F5F5F5`
- 卡片：白底，弱阴影或无阴影，圆角偏小
- 主文字：深灰黑 `#1F1F1F` / `#222222`
- 次文字：`#666666` / `#888888` / `#999999`
- 主强调色：红橙系 `#FF4A43` / `#FF3B30`
- 辅助强调色：浅红 `#FFF1F0`、浅橙 `#FFE8D9`、浅灰 `#F4F4F4`

## 布局规则

- 页面顶部优先使用白色原生小程序式标题栏，不再默认使用深色 hero
- 搜索栏应紧凑、扁平、圆角较小，贴近参考稿的电商搜索样式
- 商品列表优先使用“左图右信息”的信息流卡片
- 商品详情页优先使用“顶部标题栏 + 图集 + 价格信息块 + 规格块 + 店铺块 + 底部操作栏”
- 购物车、结算、支付结果页优先使用贴近参考稿的白底内容区和红色主 CTA
- 底部 TabBar 使用白底扁平样式，激活态以红色文本或小图标强调，不再使用深色胶囊悬浮风格

## 组件规则

- 商品卡、购物车项、订单卡、地址卡、优惠券卡保持业务组件自建
- `TDesign Miniprogram` 只用于基础控件，不主导业务卡片视觉
- 抽屉、弹层、底部操作栏、筛选面板优先采用参考稿中的轻量电商样式

## 执行顺序

1. 新页面先对齐这份文档的视觉规则
2. 再对照对应 pencil 截图确认布局和信息层级
3. 最后对照 PRD 校验功能和状态是否完整
